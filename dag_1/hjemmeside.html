HTML

HTML står for HyperText Markup Language, og er et markeringssrpåk som brukes til å formatere og strukturere nettsider.

HTML er grunnstenen i nettsider, og inneholder alt som presenteres for nettleseren; blant annet tekst, bilde-, lyd- og video-elementer.

Tags - <...>

Tags i HTML er elementer som forteller hvordan dokumentet er bygget opp.
Elementene forteller hvordan strukturen på HTML-dokumentet skal presenteres, og i et rent HTML-dokument vil alt presenteres i rekkefølge fra topp til bunn.
HTML er kort fortalt hva som skal vises på en nettside.

Alle tags avsluttes med "/>". Enten direkte (<meta author="Abelsen, Tommy"/>), eller med tekst som presenteres imellom:
(<a href="https://www.oslomet.no">OsloMet.no</a> vises som OsloMet.no)

Dokument-type - <!DOCTYPE>

!DOCTYPE forteller nettleseren hva slags type dokument som skal tolkes og presenteres.
<!DOCTYPE html> sier at dokumentet er av typen HTML, som er mest vanlig å presentere nettsider i.

HTML - <HTML>

<html>-tagen viser at alt som er innenfor denne er av typen HTML.

Head - <head>

Head-tagen inneholder viktige elementer, som for eksempel forfatter, lenker til CSS(styling)-filer og lenker til javascript-filer.
Ved å henvise til andre dokumenter (lenker til filene) slipper en å skrive all kode i HTML-dokumentet, og man kan referere til samme filer i flere dokumenter.
Dette gjør at en enkelt kan bruke samme stil og scripts på flere sider.

Det er også viktig å huske at alt det som skrives i <head> ikke blir presentert (vist) i nettleser.
Det vil si at det man skriver i <head> ikke er synlig på nettsidene, med mindre man åpner kildekoden og leser HTML.

Body - <body>

Det er først i <body> at HTML-elementer blir vist.
Alt man skriver i <body> vises på nettsiden, enten det er skrevet i tags eller som ren tekst.

Avsnitt - <p>

For å skrive tekst i HTML, bruker en gjerne "p-tags". P står for "paragraph", eller avsnitt, og brukes for å bygge opp teksten du skriver og separere avsnitt.
Det går an å skrive tekst uten å bruke tags, men dette er ikke anbefalt.

Overskrifter - <h#>

For å lage overskrifter (headlines) bruker man h-tags. H-tags går fra h1 til h6, hvor h1 er størst og h6 er minst.
Eksempler på dette:

<h1>

<h2>

<h3>

<h4>

<h5>
<h6>

Lenker - <a href>

En lenke ser slik ut:
Lenke til OsloMet.no

Man kan lage en lenke i HTML med følgende tag:
<a href="link til siden"> Tekst på lenken </a>

Klasser og ID

Klasser og id er viktige attributter som knyttes til enkelte (ID) eller hele grupper (klasser) av elementer.
Klasser og id brukes ofte i forbindelse med design og oppførsel, og gjør det lettere å lage regler for individuelle eller en gruppe elementer.

Klasser defineres ved class="navn på klasse her" i HTML tag.
Ganske likt, så defineres id slik: id="navn på id her" i en HTML

Eksempel:
<h1 class="headline"> og
<p id="content">

Det må også nevnes at alle standard-elementer i HTML er av typen klasser, så en kan enkelt gi regler til eksempelvis alle p-tags.

<div>

<div>-tagen brukes veldig ofte når man skal designe en side med en del CSS og ikke ønsker å ha forskjellige, og motstridende, regler på samme elementer.
<div> brukes gjerne sammen med klasser og id-er for å gi regler på egenlagde elementer.

I motsetning til p- og h-tags, som har veldefinerte rammer for bruk, kan <div> brukes til absolutt alt; til tekst, til bilder, til video og lyd.

CSS

CSS står for Cascading Style Sheets.
CSS brukes først og fremst til å definere utseende til HTML-, (og XML-), elementer.

CSS kan blant annet endre størrelse og farge på tekst,
samt endre posisjoner på elementer.

Der HTML er elementene som skal vises, eller skjelettet til dokumentet, er CSS hvordan elementene skal vises.
CSS definerer utseende på HTML og kan sammenlignes med hvordan muskler, hud og hår definerer utseende til et menneske.

Style

CSS kan skrives i HTML-dokumenter, både i <head> og <body>.
Man kan style elementer direkte: <p style="color:green">, eller i <style>-tags i <head>

Det er likevel mest vanlig å lage et eget stilark (stylesheet) med CSS, og linke til det arket i HTML-head.
Dette kan gjøres ved å skrive <link rel="stylesheet" type="text/css" href="filsti til .css-dokument"> i head-tagen.

Design

Design av nettsider er viktig. Både for å skape et inntrykk om en estetisk god presentasjon, og for å gjøre nettsiden enkel og oversiktlig.
Webdesign handler om hvordan skape en nettside som formidler budskapet man ønsker, eksempelvis ved å understreke eller fremheve viktige elementer, samtidig som man passer på at nettsiden skal være brukervennlig.
En lite brukervennlig nettside kan karaktiseres ved uoversiktlig plassering av elementer, distraherende eller anonymiserende fargebruk og er gjerne vanskelige å navigere.

En godt designet nettside er subjektivt, og alle har forskjellige meninger. Men likevel er det noen hovedregler man bør passe på.

Man bør tenke over:
Målet med nettsiden - skal den være informativ, skape interesse eller kanskje underholde?
Virkemidler man bruker - skal siden være fargerik, ha kontraster og forskjellige størrelser?
Brukeropplevelse - hvordan gjøre nettsiden enklest og mest mulig oversiktlig, samtidig som den har en tydelig og gjennomført stil.

Ingen av disse punktene har noen fasitsvar til hvordan designe nettsiden, men det kan bidra til en bedre designprosess.
Dette vil igjen gjøre det lettere å nå målet med god bruk av visuelle virkemidler.
